# Stratégie partenariat agent immobilier : cluster Annecy + bassin chambérien (2026)

Premier beta-testeur : **Jérémy Lange**, conseiller Capifrance, zone Annecy / Chambéry / Aix-les-Bains. Site : https://jeremy-lange.capifrance.fr/fr

Ce document fige la stratégie éditoriale du premier cluster B2B "agent immobilier partenaire" pour Vivre près de Paris. Il sert de template pour les agents suivants.

## 1. Rationale stratégique : pourquoi Annecy + bassin chambérien comme premier cluster

Trois raisons.

**Raison 1 : le keyword "vivre à Annecy" est sous-traité par les agents immo classiques.** Les SERPs sont dominés par PAP, MeilleursAgents, des blogs touristiques (annecy-pratique.fr) et 1-2 agents très locaux qui font du publi-rédactionnel transparent. Aucun acteur ne croise honnêtement "vivre à Annecy" + "alternative bassin chambérien" + "comparaison Paris / Genève / Lyon" avec un ton voix incarnée. Espace clair pour Vivre près de Paris.

**Raison 2 : la zone d'intervention de Jérémy est large et cohérente.** Annecy, Chambéry, Aix-les-Bains, Rumilly, le sillon alpin. Ça permet de produire 10 articles non redondants sans tomber dans le contenu de quartier ultra-niche qui ne scale pas. On peut faire 5 articles 100 % Annecy (qui se positionnent sur le keyword premium "vivre à Annecy") + 5 articles "bassin chambérien" qui captent un trafic alternatif moins concurrentiel (Chambéry +Aix-les-Bains restent 35-50 % moins chers qu'Annecy au m²).

**Raison 3 : l'angle Paris reste tenable.** Annecy/Chambéry sont à 3h30-3h45 TGV de Paris. Ce n'est pas une banlieue, mais c'est dans la suite logique du site "quitter Paris pour la qualité de vie". Pour un cadre télétravail 4j/5, ça reste accessible. Le site n'a pas besoin de pivoter.

## 2. Maillage interne : 1 pillar + 9 satellites

L'article pillar du cluster est **"Pourquoi vivre à Annecy en 2026 : le bilan honnête"** (`pourquoi-vivre-annecy-2026`). Il sert d'ancre. Les 9 autres pointent vers lui dès qu'ils mentionnent Annecy en général.

Maillage par paires logiques (10 articles : 5 Annecy + 5 bassin) :
- `pourquoi-vivre-annecy-2026` (pillar) ↔ `quartiers-annecy-ou-vivre-2026` (satellite zoom quartiers)
- `quartiers-annecy-ou-vivre-2026` ↔ `annecy-familles-ecoles-nature-securite` (école par quartier)
- `prix-immobilier-annecy-2026-qui-achete` ↔ `annecy-chambery-immobilier-ou-acheter-2026` (comparaison budget)
- `teletravail-annecy-vs-geneve-lyon` ↔ `pourquoi-vivre-annecy-2026` (angle pro pillar)
- `annecy-familles-ecoles-nature-securite` ↔ `pourquoi-vivre-annecy-2026` (angle famille pillar)
- `vivre-annecy-travailler-chambery-faisable` ↔ `triangle-annecy-chambery-aix-les-bains` (vie pro distribuée)
- `triangle-annecy-chambery-aix-les-bains` ↔ tous les articles bassin
- `annecy-chambery-immobilier-ou-acheter-2026` ↔ `bassin-chamberien-alternative-budget`
- `bassin-chamberien-alternative-budget` ↔ `tgv-paris-annecy-chambery-mobilite-2026`
- `tgv-paris-annecy-chambery-mobilite-2026` ↔ `pourquoi-vivre-annecy-2026` (boucle de fermeture)

## 3. Anti-marketing : qu'est-ce qui fait que ce n'est pas du publi-rédactionnel agent immo

Cinq garde-fous.

**a. Le ton voix Vivre près de Paris.** Tutoiement, voix incarnée ("j'ai vu", "soyons clairs"), zéro em-dash, banlist phrases-tells respectée. Aucune phrase d'agent immo standard ("Annecy est une ville dynamique offrant un cadre de vie exceptionnel" → bannie).

**b. Chiffres précis et défavorables quand nécessaire.** Annecy 5 930 €/m² appartement vs Chambéry 3 431 €/m². On dit ce qui pique : Annecy s'est embourgeoisée, marché tendu, tourisme estival agressif. Un publi-rédactionnel n'écrirait jamais "ce qui plante" honnêtement.

**c. Le CTA Jérémy est en aside ou en bas, jamais en H2.** Le contenu informatif passe d'abord. L'agent vient en option pour qui veut creuser. Le user qui lit ne se sent pas pressuré.

**d. La structure varie par article.** Comparaison, FAQ, vignettes, top, récit-enquête. Pas 10 articles "intro + 7 H2 + conclusion".

**e. Le positionnement Jérémy = "l'expert local cité", pas "le vendeur".** Phrasing type : "Jérémy Lange (Capifrance), qui couvre la zone Annecy-Chambéry-Aix-les-Bains depuis plusieurs années, m'a confirmé que..." → on le présente comme source, pas comme partenaire commercial.

## 4. Template réutilisable pour agents suivants

Pour chaque agent partenaire futur (autre ville, autre agent), répliquer ce gabarit :

**Étape 1 : Briefing agent (45 min).** Récupérer zones précises, années d'expérience, spécialisations (familles ? investisseurs ? primo-accédants ?), 2-3 quartiers où il a vendu récemment, 1-2 anecdotes terrain réutilisables.

**Étape 2 : Stratégie 10 articles.** Découper en 5 + 5 :
- 5 articles 100 % ville principale (pillar "pourquoi vivre à X" + 4 satellites thématiques)
- 5 articles bassin élargi (positionnement de la ville principale par rapport à 2-3 villes voisines complémentaires)

**Étape 3 : Briefs SEO.** Format identique à `lib/blog-posts.ts`. Audience/angle/ouverture/structure/sections/references. Intro 200 mots prête.

**Étape 4 : CTA varié.** 10 formules différentes, jamais le même bloc copy-collé. Variation : aside milieu d'article, encart bas d'article, mention de source. L'agent doit être présent dans les 10 articles sans devenir lassant.

**Étape 5 : Mesure 90 jours.** Trafic, scroll depth, clics sur CTA agent, lead-form filled chez l'agent. Si l'agent confirme leads, le modèle scale. Sinon, ajustement angle ou wording CTA.

**Coût template estimé par agent** : 10 briefs × 1h rédaction = 10h + 10 articles générés via `regenerate-blog` (cost API Opus 4.7) + 0,5 jour mise en page / vérification. ~15h homme pour un cluster complet, hors maintenance trimestrielle.

## 6. Notes techniques

- Annecy / Chambéry / Aix-les-Bains n'existent pas encore dans `SAMPLE_COMMUNES` (`lib/sample-data.ts`). Le user les ajoute en parallèle. Les briefs mentionnent les villes par leur nom mais n'utilisent pas de slug interne `/communes/annecy` tant que la commune n'est pas en base.
- Les 10 articles ont une `publishedAt` étalée entre 2026-04-26 et 2026-05-26 (pas de futur).
- La catégorie est mappée sur l'existant : `guide` pour les articles vie quotidienne, `tendance` pour immobilier prospectif, `finance` pour budget pur, `persona` pour profils ciblés, `transport` pour TGV/mobilité.
- Aucun em-dash ajouté dans les 10 briefs. Baseline avant : 31 dans lib/blog-posts.ts. Vérification avec `grep` sur le caractère em-dash dans le fichier.
